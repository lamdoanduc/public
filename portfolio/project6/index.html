The impact of marketing campaigns on the total app trial traffic.

June 11, 2021

Goal:

A company has been running paid marketing on various platforms. The company wants to understand the true impact of the marketing campaigns on their total new trial traffic. This data project is aimed to:

  • Analyse the data to see the number of new users and main contributing factors;
  • Suggest for how to best allocate a marketing budget 50 000 USD over the next marketing period (28 days).

Data:

The data from the last year includes two datasets. One contains daily data of app trials per day, and the other contains ad spending by day and campaign.

Method and workflow:

  • First, an exploratory data analysis was carried out to get insights from the data.
  • The importance metric to see the effect of the marketing campaigns is cost per trail, e.g., an effective marketing campaign is the one with a low cost per trial. There were several marking campaigns (on different platforms and target audience) run each day in the entire period. However, with the given data it was impossible to track the traffic of new trails, i.e., where (which marketing campaign) the new trials were originated. The most (least) effective campaigns were determined by using rigorous analyses of the cost per trail by day and the correlation between the marketing spend and the number of trials.
  • To suggest for marketing budget allocation for the next marketing period, I used Linear Programming (optimisation) method. It is a 2-step process:
    • First, linear regression: Use a linear regression model to fit the spending amount of the campaigns and the cost_per_trail. The 14 variates are the spending amount of the 14 unique campaigns.
    • Then, Linear Programming or linear optimization: to suggest the budget allocation for each day in a period of 4 coming weeks. We will be minimising the cost_per_trial for each day. The constraints for the optimisation problem are:
      • Assume that the budget of 50 000 $ is equally divided into the same amount for each day in the next period.
      • The budget allocation will depend on the number of campaigns (a variable n_campaign) we want to launch each day.
      • For a given date, the amount allocated for each campaign depends on the data collected from the beginning to the date before. This means we need to re-run the code every day with the data accumulated to get the budget allocation for the date.

Outlook

  • The data does not include the customer journey. The analysis would be improved with the supplemented data.
  • The process can be constructed as a pipeline where the code connected to the accumulated data and to be scheduled to run regularly. It can be daily scheduled to get the optimised budget allocation.
